# ⚔ Life Quest TUI

> Геймифицированный менеджер задач и помодоро-таймер прямо в вашем терминале.
> Написан на Rust с использованием [ratatui](https://github.com/ratatui-org/ratatui).

## Возможности

- **Задачи с XP** — каждая задача приносит очки опыта при выполнении. Размер награды рассчитывается автоматически исходя из длины описания задачи (более подробные задачи — больше XP, максимум 500 XP за задачу).
- **Уровни** — при накоплении 100 × текущий_уровень XP происходит повышение уровня с праздничным оверлеем прямо в терминале.
- **Помодоро-таймер** — цикл 50 минут работы / 10 минут перерыва. После каждого завершённого рабочего блока автоматически начинается перерыв и начисляется +50 XP.
- **Напоминания** — каждые 30 минут во весь экран появляется окно с вопросом «Что ты сейчас делаешь?». Если отвечаешь `Y` — получаешь +5 XP. Если `N` — программа предлагает случайную незакрытую задачу.
- **Системные уведомления** — при завершении помодоро и напоминаниях отправляются нативные уведомления через `notify-send` (Linux) или `osascript` (macOS).
- **Сохранение прогресса** — все задачи, XP, уровень и счётчик помодоро сохраняются в файл `life_bot_save.json` рядом с бинарником и автоматически загружаются при следующем запуске.
- **Русская раскладка** — горячие клавиши работают как на латинской, так и на русской ЙЦУКЕН-раскладке. Переключать язык ввода не нужно.

## Установка и запуск

### Требования

- Rust 1.75 или новее (`rustup update stable`)
- Терминал с поддержкой Unicode и 256 цветов (практически любой современный: iTerm2, Alacritty, Kitty, GNOME Terminal, Windows Terminal)
- Linux: пакет `libnotify` / утилита `notify-send` для системных уведомлений (опционально)

### Сборка из исходников

```bash
git clone https://github.com/yourname/life-quest
cd life-quest
cargo build --release
./target/release/life_quest_ratatui
```

### Запуск в режиме разработки

```bash
cargo run
```

Сохранение создаётся в текущей рабочей директории (`./life_bot_save.json`). Если хотите хранить прогресс в конкретном месте — запускайте бинарник из нужной папки или сделайте симлинк.

## Управление

### Главный экран

| Клавиша | Действие |
|---|---|
| `A` | Открыть диалог добавления задачи |
| `1` — `9` | Отметить задачу под соответствующим номером как выполненную |
| `S` | Запустить таймер (только из состояния Idle) |
| `P` | Пауза / продолжить таймер |
| `R` | Сбросить таймер в начальное состояние |
| `M` | Переключить режим таймера (Работа ↔ Перерыв) |
| `Q` | Выйти из приложения |
| `Ctrl+C` | Принудительный выход |

### Диалог добавления задачи

| Клавиша | Действие |
|---|---|
| Любой символ | Ввод текста задачи |
| `Backspace` | Удалить последний символ |
| `Enter` | Сохранить задачу и вернуться |
| `Esc` | Отменить, вернуться на главный экран |

### Окно напоминания

| Клавиша | Действие |
|---|---|
| `Y` | «Да, работаю!» — +5 XP, закрыть окно |
| `N` | «Ничего не делаю» — показать рекомендацию из задач |
| `Esc` | Закрыть окно без начисления XP |

### Оверлей «Level Up!»

Появляется автоматически при повышении уровня. Закрывается нажатием `Пробел`.

> Все горячие клавиши работают и в русской раскладке: `Ф` = `A`, `Ц` = `S`, `К` = `R` и т.д.

## Как работает таймер

`Timer` хранит `remaining` — оставшиеся секунды — и `rem_start` — значение `remaining` в момент последнего `start()` или `resume()`. Вместо убывающего счётчика используется `Instant::now()`: метод `update()` при каждом вызове вычисляет `elapsed = start.elapsed().as_secs()` и пересчитывает `remaining = rem_start - elapsed`. Это исключает накопление ошибок от цикла событий и позволяет делать паузы без потери точности.

Цикл событий опрашивает таймер с интервалом 200 мс (`event::poll(Duration::from_millis(200))`). Когда `update()` возвращает `true` (время вышло), приложение автоматически переключает режим и запускает следующую фазу.

## Как считается XP за задачу

```rust
let xp = ((text.len() as u32 / 5) + 1).min(50) * 10;
```

Каждые 5 символов в описании задачи дают +1 к множителю, минимум — 1, максимум — 50. Итоговый XP = множитель × 10. Диапазон: от **10 XP** (пустое/очень короткое описание) до **500 XP** (описание длиной 245+ символов).

## Система уровней

Порог XP для перехода с уровня N на уровень N+1 равен `N * 100`. То есть:

- Уровень 1 → 2: 100 XP
- Уровень 2 → 3: 200 XP
- Уровень 3 → 4: 300 XP
- ...

При переходе лишний XP не теряется — остаток переносится на следующий уровень.

## Сохранение

Прогресс хранится в `life_bot_save.json` в виде простого JSON-объекта:

```json
{
  "tasks": [
    { "id": 1, "text": "Написать тесты", "done": false, "xp": 30 }
  ],
  "xp": 75,
  "level": 2,
  "pomodoros_done": 4,
  "next_task_id": 2
}
```

Файл читается при старте и перезаписывается при каждом изменении состояния. Если файл повреждён или отсутствует, создаётся чистое состояние без ошибок.

## Отличия от оригинальной версии (crossterm)

Оригинал использовал `crossterm` напрямую: ручное позиционирование курсора через `cursor::MoveTo`, буферизацию через `queue!` и посимвольную отрисовку каждого элемента. Версия на ratatui полностью декларативна:

- **Layout** описывается через `Layout::vertical([Constraint::Length(n), ...])` — терминал автоматически разбивается на зоны без магических координат.
- **Виджеты** (`Paragraph`, `List`, `Block`) принимают данные и стили, сами рассчитывают позиции символов.
- **Diff-рендеринг** — ratatui сравнивает текущий и предыдущий буфер и перерисовывает только изменившиеся ячейки, что снижает мигание.
- **Попапы** реализованы через `Clear` + рендер поверх: `f.render_widget(Clear, area)` стирает фон, затем поверх накладывается `Paragraph` с `Block`.
- Единственное место, где crossterm остался напрямую — обработка событий клавиатуры (`event::poll` / `event::read`), что является стандартной практикой при работе с ratatui.
